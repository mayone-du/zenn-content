### Rust で Web フロントを書く際の選択肢

  - https://github.com/yewstack/yew

  - https://github.com/seed-rs/seed

自分が聞いたことあるのだとこのくらいです。`Yew`が`18k`と一番スター数が多かったので、`Yew`を採用しました。

## 今回作ってみた感想や得られた知識など

- WASM を生成しているということを忘れちゃいけない
  - `std::fs`を使おうとしてコンパイルは通るけど、ランタイムで動かなくて詰まった。(結局日本の Rust コミュニティで解決していただいた。詳しくは自分のスクラップへ。)
- Rust の Web フロントのエコシステムはまだまだ貧弱。個人的には、本気で Web アプリ作るとしたら Rust でやるメリットはそこまでないように感じた。主にエコシステムや、慣れの面で。

#### できたこと

- ルーティングや HTTP リクエスト、state 管理などの基本的なこと
- Tailwind CSS でのスタイリング
- GitHub Pages / GitHub Actions での自動デプロイ

#### できていないこと

- トップページ以外へ直接アクセスした際のルーティング
- API リクエスト時のキャッシュ管理
- グローバルステート(今回は必要なかったから、そもそも調べてない)

## まとめ

- 1 から Rust でやるのはまだまだしんどい(色んな意味で)
- ただ、個人的には好きだしこれからも追い続けたい。書いてて楽しいし、なんか俺かっけー感あって良い(←)
- よく言われているように、使う場面はおもい処理を Rust で書くみたいな感じになりそう。すぐに Rust に置き換わるとかはまぁ無いだろうなって思った。
